# 变更：重构前后端分离架构并升级前端应用

## 原因
当前项目采用 Python 标准库同时托管静态页面与 API，请求协议、前端渲染逻辑和后端扫描结果强耦合，不利于前端重构、公网部署和后续接口演进。为了支持 React + Vite 前端重构、独立 API 服务和更稳定的数据模型，需要对前后端边界、接口契约和部署结构进行系统性调整。

## 变更内容
- 新增独立后端 API 服务边界，逐步替代 `serve_media_wall.py` 的静态托管 + API 混合模式。
- 将现有海报墙扫描、缓存、TMDb 聚合逻辑按 `config / repository / scanner / service / api` 分层重构。
- 重设计前端消费的数据契约，拆分分类树、媒体列表、媒体详情等 API DTO，避免当前胖响应直接暴露给前端。
- 新建基于 React + Vite 的独立前端应用，承接分类浏览、媒体列表、详情展示和后续管理能力扩展。
- 为公网部署预留运行边界，包括统一 API 前缀、前后端分离部署方式、基础鉴权入口和配置脱敏策略。

## 影响
- **受影响的规范**：海报墙前后端架构、媒体分类浏览、媒体详情展示、分类刷新与缓存管理、部署方式。
- **受影响的代码**：
    - `serve_media_wall.py`: 现有静态托管与 API 混合入口将被替换或降级为兼容入口。
    - `media_wall_service.py`: 需要拆分扫描、缓存、配置和仓储职责，并为新 API 提供稳定服务层。
    - `media_wall_site/app.js`: 现有原生脚本前端将被 React 前端逐步替代。
    - `media_wall_site/index.html`: 现有静态单页入口将不再作为长期主前端入口。
    - `config.yml`: 需要适配前后端分离后的服务配置、部署配置和敏感信息治理。
    - `openlist_sdk/client.py`: 继续作为后端基础设施层复用，供独立 API 服务调用。
    - `tmdb_sdk.py`: 继续作为元数据基础设施层复用，供独立 API 服务调用。
