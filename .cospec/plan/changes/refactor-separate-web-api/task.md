- [x] 1.1 新增独立后端 API 服务目录与启动入口
     【目标对象】`backend/__init__.py`、`backend/app.py`、`backend/main.py`、`serve_media_wall.py`
     【修改目的】建立可独立部署的后端 API 服务入口，并明确旧混合入口的兼容降级方式，替代当前静态托管与 API 混合模式。
     【修改方式】新增 `backend` 应用骨架；在 `backend/app.py` 中新增应用工厂或应用实例组装逻辑；在 `backend/main.py` 中新增独立启动入口；在 `serve_media_wall.py` 中修改启动职责，保留兼容转发、兼容提示或退役说明之一，不再继续承载长期主 API 入口。
     【相关依赖】`config.yml`、`media_wall_service.py`
     【修改内容】
        - 新增独立后端目录结构，并按仓库现有 Python 模块命名风格组织 `config / repository / scanner / service / api / dto` 分层。
        - 在后端入口中统一挂载 `/api/v1` 前缀，补充健康检查路由，并为后续管理员鉴权预留中间件、依赖注入或校验入口。
        - 明确开发环境与生产环境的启动方式差异，包括本地调试入口、部署启动入口与旧脚本兼容入口的职责边界。
        - 为旧入口增加兼容处理策略：若继续保留则仅承担兼容转发或提示；若降级则明确不可继续新增业务逻辑，避免双入口并行演进。
        - 补充启动失败时的错误处理要求，包括配置缺失、依赖初始化失败、端口或服务启动异常时输出与现有项目一致的错误信息。
- [x] 1.2 拆分后端核心模块并沉淀稳定服务边界
     【目标对象】`media_wall_service.py`、`backend/config/`、`backend/repository/`、`backend/scanner/`、`backend/service/`
     【修改目的】将当前单文件中的配置、缓存、扫描、聚合、服务职责拆分为可维护模块，为独立 API 服务提供稳定服务层。
     【修改方式】修改 `media_wall_service.py` 的职责边界，仅保留兼容编排或迁移过渡逻辑；新增配置层、仓储层、扫描层、服务层模块；按现有实现拆分并迁移相关函数、类和数据组织方式。
     【相关依赖】`config_loader.py`、`openlist_sdk/client.py`、`tmdb_sdk.py`、`media_wall.db`
     【修改内容】
        - 从现有单文件中抽离配置解析与默认值处理，确保新配置层负责读取、校验和向服务层提供结构化配置。
        - 抽离 SQLite 缓存访问职责到仓储层，统一负责媒体索引、分类缓存和刷新写入，避免路由层或扫描层直接操作数据库。
        - 抽离 OpenList 扫描与 TMDb 聚合职责到 scanner 层，明确扫描成功、元数据缺失、第三方接口失败等分支处理。
        - 在 service 层封装分类树、媒体列表、媒体详情、刷新编排等稳定调用接口，避免 API 层直接依赖底层扫描或数据库细节。
        - 迁移过程中保持现有字段命名、日志和异常处理风格，不额外发明新的跨层调用模式；对暂未迁移完的兼容逻辑明确过渡边界。
